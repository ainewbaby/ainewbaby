Object Detection

发表于 2017-09-25

Rich feature hierarchies for accurate object detection and semantic segmentation

R-CNN两个关键点

  1. 用CNN生成region proposals
  2. 先在IMAGENET进行预训练，再在PASCAL上进行fine-tuning

传统做法，如SIFT、HOG为梯度直方图，类似于视觉系统中的第一层V1。

早期用于物体检测的CNN，为了保持较高的空间分辨率，通常只有两个卷积和池化层。

作者本想使用扫窗（类似OverFeat做法），但经过五个卷积层后感受野很大（195x195），stride也很大（32x32）。

因此决定采用“recognition using regions”的范式。

R-CNN分三步：

  1. 用selective search生成2000个类别无关的region proposals
  2. 用CNN对每个region提固定长度的特征向量
  3. 对每类分别用一个线性SVM进行二分类，对每一类用NMS拒绝那些IoU得分大于阈值的region

region proposals IoU大于等于0.5的为正样本，否则为负样本。

128 mini-batch = 32正样本 + 96背景（因为一张图像中正样本相对较少）

看到了2.5

R-FCN: Object Detection via Region-based Fully Convolutional Networks

计算共享

单帧170ms

分类与检测的矛盾：

  • 分类：希望图像具备平移不变性，即无论图像中的物体如何变化都能被识别
  • 检测：希望图像具备平移可变性，即候选框随着图像中物体的平移而进行相应的调整

网络越深，对图像的平移越不敏感。

ResNet论文中为了解决这个矛盾，引入RoI pooling层，这种region-specific的操作破坏了平移不变性，使得后续的卷积层不再具备平移不变性。

two-stage策略

  1. region proposal
  2. region classification

骨架

  • 基于ResNet-101：100个卷积层+GAP+1个全连接层
  • 去除GAP和FC层
  • ResNet101的最后一个卷积层是2048维，我们接上一个1024维的1x1卷积来降维
  • 最后，我们使用k^2*(C+1)通道数的卷积层来生成score maps

训练

  • 损失函数分为两部分：分类损失（交叉熵）与bbox回归损失
  • IoU大于等于0.5为正样本，小于0.5为负样本
  • OHEM
    • 假设每张图像有N个proposals，正向传播时需计算N个proposals的损失
    • 对这些RoIs按损失大小进行排序，选出B个损失最高的RoIs
    • 反向传播时只对选出的B个RoIs进行
  • 默认使用single-scale进行训练，对图像进行resize（600 px）

测试

  • 提完特征经过PRN生成300个RoIs
  • 后处理：非极大值抑制（0.3 IoU）

ResNet101的stride由32变为16，增加了score map的分辨率，conv4以前的stride保持不变，conv5由2变为1，同时引入带孔卷积，保持相同的感受野。

对比试验的部分结论

  • 生成的候选框数目太多也没用，300足以
  • OHEM对于R-FCN没有占用太多时间
  • 多尺度训练{400， 500， 600， 700， 800}
  • 深度影响：50 < 101 约等于152
  • R-FCN对小物体检测更好
